title: Deploy Gateway with Kubernetes
---

## Overview

Conduktor provides a [Helm repository](https://helm.conduktor.io) containing a chart that will deploy Conduktor Gateway on your Kubernetes cluster. We don't provide any Kafka cluster dependency.

For a full reference of our recommended Conduktor Platform deployment architecture, [see our sample GitHub repository](https://github.com/conduktor/conduktor-reference-architecture).

## General requirements

* A Kubernetes cluster and Kubectl ([install](https://kubernetes.io/docs/tasks/tools/#kubectl))
* Helm 3.1.0+ ([install](https://helm.sh/docs/intro/install/))
* A Kafka cluster reachable from your Kubernetes cluster


## 1. Set up Helm repository

```shell
helm repo add conduktor https://helm.conduktor.io
helm repo update
```

## 2. Configure helm values

Copy the following template into a file called **values.yaml**. Full reference in our [Git repository](https://github.com/conduktor/conduktor-public-charts/blob/main/charts/gateway/values.yaml).

```yaml title="values.yaml"
gateway:
  replicas: 1
  env: 
    KAFKA_BOOTSTRAP_SERVERS: my-cluster-kafka-bootstrap:9092
```

See more on how to [connect to your Kafka cluster](/guide/conduktor-in-production/deploy-artifacts/deploy-gateway#3-connect-gateway-to-kafka).

## 3. Install the chart

```shell
helm install mygateway conduktor/conduktor-gateway -f values.yaml
```
## 4. Produce and Consumer through the Gateway

```shell
kubectl -n kafka run kafka-producer -ti --image=quay.io/strimzi/kafka:0.47.0-kafka-4.0.0 --rm=true --restart=Never -- bin/kafka-console-producer.sh --bootstrap-server gateway-conduktor-gateway-internal:9092  --topic my-topic
```

```shell
kubectl -n kafka run kafka-consumer -ti --image=quay.io/strimzi/kafka:0.47.0-kafka-4.0.0 --rm=true --restart=Never -- bin/kafka-console-consumer.sh --bootstrap-server my-cluster-kafka-bootstrap:9092 --topic my-topic --from-beginning
```

## Compatibility matrix

This compatibility matrix is a resource to help you find which versions of Conduktor Gateway work on which version of our Conduktor Gateway Helm chart.

> We recommend you use the version of Gateway that comes pre-configured with the Helm chart. You can adjust the version in your values property according to the supported Gateway version, if required.
> Breaking changes column only lists a **breaking change in the Helm chart**. See Conduktor [release notes](https://docs.conduktor.io/changelog/) to determine whether there are breaking changes within the artifacts.
